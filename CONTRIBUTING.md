# Contributing to SozoRock Health

## Scope and safety

Contributions must preserve the platform's non-clinical boundary, national discovery model, state-aware provider readiness, resident-data minimization, and de-identified CB-CAP rules. Do not add real resident data, provider credentials, secrets, or production configuration to the repository.

## Working agreement

1. Start from an issue or an approved work item with a clear user outcome.
2. Keep changes small, reviewable, and covered by automated checks.
3. Update the relevant product, privacy, API, or operational documentation with behavior changes.
4. Treat accessibility, language access, and low-connectivity support as acceptance criteria.
5. Do not weaken privacy thresholds, state-licensure checks, feature controls, or release gates to make a check pass.

## Pull requests

Each pull request must describe the user outcome, affected roles, privacy/data impact, accessibility impact, AI/feature-control impact, test evidence, and rollback consideration. It must pass the required automated checks before merge.

## Reporting security concerns

Do not report a security concern in a public issue. Follow [SECURITY.md](SECURITY.md).
